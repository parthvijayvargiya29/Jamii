🧱 PART 1 — Database schema (PostgreSQL)

Create the following tables only if they do not already exist.

1️⃣ User availability table

This stores when staff are available to work.

user_availability
- id (PK)
- user_id (FK → users.id)
- restaurant_id (FK → restaurants.id)
- day_of_week (INTEGER, 0–6)
- start_time (TIME)
- end_time (TIME)
- is_available (BOOLEAN, default true)
- created_at (TIMESTAMP)


Each user can have multiple availability records.

2️⃣ Shifts table

This stores planned shifts.

shifts
- id (PK)
- restaurant_id (FK → restaurants.id)
- shift_date (DATE)
- start_time (TIME)
- end_time (TIME)
- station (TEXT)
- required_staff (INTEGER)
- created_at (TIMESTAMP)

3️⃣ Shift assignments table

This links users to shifts.

shift_assignments
- id (PK)
- shift_id (FK → shifts.id)
- user_id (FK → users.id)
- status (TEXT: assigned / confirmed / swapped)
- created_at (TIMESTAMP)

🔌 PART 2 — Backend API routes (Express)

Add the following routes:

Availability

POST /availability → create or update user availability

GET /availability/:userId → fetch availability for a user

Shifts

POST /shifts → create a shift

GET /shifts?date= → fetch shifts for a specific day

GET /shifts/week?start= → fetch shifts for a week

GET /shifts/month?year=&month= → fetch shifts for a month

Assignments

POST /shift-assignments → assign a user to a shift

Prevent double-booking (no overlapping shifts for the same user)
